Política de Privacidad de ALERTA

Esta Política de Privacidad describe cómo la aplicación ALERTA (paquete: com.example.alerta; en inglés puede mostrarse como ALERT) recopila, usa, almacena y protege la información cuando usted utiliza nuestros servicios.

Nota sobre esta versión: esta es una versión de prueba para testers. Todas las funciones están desbloqueadas. No incluye compras dentro de la aplicación ni procesamiento de pagos. No utiliza Google Play Billing.

2. Descripción general

ALERTA es una aplicación para dispositivos Android que monitoriza eventos del sistema y reproduce alertas por voz (texto a voz / TTS) cuando detecta cambios relevantes configurados por el usuario, incluyendo de forma no exhaustiva:

  • Conexión o desconexión de Wi-Fi, datos móviles o Ethernet.
  • Estado del cargador y nivel de batería.
  • Temperatura del dispositivo (núcleos del procesador y/o batería, cuando el sistema lo permite).
  • Uso de memoria RAM y espacio libre de almacenamiento.
  • Consumo de datos móviles (si el usuario activa esta alerta).
  • Alertas programadas personalizadas (hasta 10, con texto y hora definidos por el usuario).
  • Anuncio de la hora en punto (si el usuario lo activa).

3. Datos que utiliza la aplicación

ALERTA está diseñada para funcionar principalmente de forma local en el dispositivo del usuario. La App puede acceder a información del dispositivo necesaria para su funcionamiento:

  • Estado de conectividad (Wi-Fi, datos móviles, Ethernet u otros tipos de red).
  • Estado de carga y nivel de batería.
  • Información térmica del dispositivo (temperatura de núcleos y/o batería).
  • Uso de memoria RAM y espacio libre de almacenamiento interno.
  • Consumo de datos móviles en tiempo real (solo si el usuario activa el monitoreo).
  • Datos del acelerómetro (solo para detectar agitación del dispositivo y pausar temporalmente las alertas de voz, si el usuario activa esta función).
  • Preferencias del usuario: idioma, tipos de alerta, umbrales, voces TTS, sensibilidad de agitación, intervalos de monitoreo, etc.

Esta información se procesa en el dispositivo para generar alertas, notificaciones y mensajes de voz. ALERTA no vende datos personales ni los utiliza con fines publicitarios.

4. Permisos del sistema Android

Para ofrecer sus funciones, ALERTA puede solicitar permisos del sistema, tales como:

  • Red e Internet: detectar cambios de conectividad y compatibilidad del dispositivo. La App no transmite datos personales a servidores del desarrollador.
  • Wi-Fi y estado de red: avisar cuando cambia la conexión.
  • Estado del teléfono (READ_PHONE_STATE): detectar cambios de conectividad móvil; no se usa para identificar al usuario, realizar llamadas ni crear perfiles.
  • Servicio en primer plano: mantener el monitoreo activo con notificación visible al usuario.
  • Notificaciones: informar del estado del servicio y alertas (Android 13+).
  • Audio y vibración: reproducir alertas por voz y avisos hápticos.
  • Inicio automático: reiniciar el servicio tras encender el dispositivo, si el usuario lo habilita.
  • Optimización de batería: solicitar al usuario excluir la App de restricciones, si es necesario para el monitoreo en segundo plano.
  • Almacenamiento: solo si fuera necesario por compatibilidad con versiones antiguas de Android o detección térmica del sistema.
  • Permisos adicionales de compatibilidad: algunos permisos pueden declararse en el manifiesto de la App por compatibilidad con distintos fabricantes y versiones de Android, sin usarse para rastrear al usuario.

Ubicación: ALERTA no utiliza la ubicación GPS para rastrear al usuario ni para crear perfiles de localización. Si algún permiso relacionado con ubicación aparece en el sistema, no se emplea para identificar la posición del usuario.

5. Servicio en primer plano

ALERTA utiliza un servicio en primer plano para mantener activo el monitoreo en segundo plano y ofrecer alertas oportunas. El usuario verá una notificación persistente mientras el servicio esté activo. Esto es necesario para el funcionamiento principal de la App.

6. Almacenamiento local y copias de seguridad

Las preferencias del usuario y la configuración de la App se guardan localmente en el dispositivo (por ejemplo, mediante almacenamiento interno del sistema / SharedPreferences).

En esta versión de prueba no se gestionan compras, pagos ni suscripciones dentro de la aplicación.

Si el usuario activa copias de seguridad del sistema en Android, los datos locales de configuración de la App podrían incluirse en esa copia, según la configuración del dispositivo y del fabricante.

7. Texto a voz (TTS)

Las alertas por voz utilizan el motor de texto a voz (TTS) instalado en el dispositivo del usuario (servicios del sistema o de Google, según configuración). ALERTA no envía el contenido de las alertas a servidores propios del desarrollador; la síntesis de voz se realiza en el dispositivo o mediante el proveedor TTS configurado por el sistema.

8. Recopilación, uso y finalidad de los datos

  • ALERTA no recopila, vende ni comparte datos personales de forma remota con fines comerciales o publicitarios.
  • ALERTA no muestra publicidad ni utiliza datos para segmentación publicitaria.
  • Toda la configuración y los datos de monitoreo se procesan y almacenan localmente en el dispositivo.
  • La finalidad del tratamiento es exclusivamente prestar el servicio de alertas solicitado por el usuario.

9. Compartición con terceros

ALERTA no comparte datos personales con terceros con fines comerciales o publicitarios. La App puede interactuar únicamente con:

  • Servicios del sistema Android (red, batería, sensores, TTS, notificaciones).
  • Google Play para distribución e instalación de la App desde la tienda (en pruebas cerradas o producción).

No existen servidores propios del desarrollador que reciban datos de los usuarios.

10. Transferencias internacionales

ALERTA no opera servidores propios para recopilar datos personales. Si el usuario utiliza servicios de Google (Play Store, motor TTS, etc.), dichos servicios pueden estar sujetos a las políticas de privacidad de Google.

11. Seguridad

Se aplican medidas razonables para proteger la información procesada localmente en el dispositivo. Dado que la App no transmite datos personales a servidores del desarrollador, el riesgo de exposición remota de datos por parte de ALERTA es limitado.

12. Conservación de los datos

La configuración y los datos locales se conservan en el dispositivo hasta que el usuario:

  • Modifica o desactiva funciones desde la App.
  • Desinstala la App.
  • Borra los datos de la aplicación desde los ajustes del sistema Android.

13. Derechos del usuario

El usuario puede en cualquier momento:

  • Desactivar alertas y el servicio desde la configuración de la App.
  • Revocar permisos desde los Ajustes → Aplicaciones → ALERTA → Permisos de Android.
  • Desinstalar la App, lo que elimina los datos locales asociados (salvo copias de seguridad del sistema, si las hubiera).
  • Agitar el dispositivo para silenciar alertas temporalmente (~30 segundos), si activó esa función.
